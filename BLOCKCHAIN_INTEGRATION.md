# ⛓️ Blockchain Integration Mechanics

This document breaks down the core architecture of how this project connects traditional Web2 E-Commerce (PHP/MySQL) directly to the Web3 Ethereum Blockchain.

## 🤔 How is the E-Commerce Store Linked to the Blockchain?

Most websites use Visa, Stripe, Stripe, or PayPal. When a payment is processed, the website asks the Centralized Bank to confirm it.
In this **Hybrid Architecture**, the website acts completely decentralised when it comes to money. We link the two worlds via `Ethers.js` and a **Transaction Hash (`bx_txid`)**.

1. **User Creates Order**: The PHP backend creates a normal cart checkout sequence and calculates a localized fiat-to-Ethereum price conversion. The order gets temporarily placed into the SQL database with a status of **Pending**.
2. **Blockchain Prompt**: When the user selects "Blockchain / Crypto", they are thrown into a localized JavaScript space (`blockchain-payment.php`). The browser's `ethers.js` script asks the MetaMask extension to open up.
3. **Execution**: The user clicks 'Confirm' in MetaMask. A direct connection to the Smart Contract on the local node (`Hardhat`) is made.
4. **Permanent Record**: The Smart Contract records the payment immutably with the specific `Order ID`. It guarantees that nobody can delete the proof that this person paid.
5. **Database Anchoring**: Once the JavaScript gets a network receipt from the Blockchain that the money successfully landed, it grabs the immutable `Transaction Hash` generated by the transaction block and passes it back to PHP (`update-blockchain-status.php`) via AJAX.
6. **Delivery**: The traditional PHP server updates the status in the MySQL database from **Pending** to **Verified**, permanently storing the Blockchain Hash inside the order history file for admins to see.

## 🛠️ The Tech Stack Inside the Blockchain

Here are the precise tools configuring the Web3 side of the project:

- **Ethereum Smart Contract (`Solidity ^0.8.0`)**: The programming logic executed autonomously inside the blockchain that holds crypto funds and issues the receipt events.
- **Node.js (`Hardhat Runtime`)**: The testing framework running on `http://127.0.0.1:8545`. It simulates the massive Ethereum network flawlessly on just your laptop so you can deploy your contracts without waiting for global block validation.
- **Ethers.js (`v6`)**: This is the translation toolkit. Instead of learning confusing base RPC API commands to talk to the block, `Ethers` wraps it all naturally in JavaScript inside the browser allowing direct `await contract.payForOrder(...)` executions.
- **MetaMask (`Chrome Extension`)**: Acts as your digital passport and signature. Web browsers cannot natively sign cryptographic hashes to spend Ethereum. MetaMask bridges this gap dynamically.
